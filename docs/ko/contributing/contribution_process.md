# 기여 프로세스

## 1. 기능 또는 개선 사항 승인 받기

GitHub 이슈를 생성하고 추천(upvote)이 있는지 확인하세요. 기능이 충분히 가치 있고 저장소에 기여하고 싶다면, [Yuhong](https://github.com/yuhongsun96)을 태그하여 검토를 요청하세요.

1주일 내에 응답이 없으면 이슈를 포함하여 yuhong@onyx.app으로 이메일을 보내도 됩니다.

모든 소규모 기능과 개선 사항이 수락되는 것은 아닙니다. 기능 풍부함과 비대화(bloat) 사이의 균형이 있기 때문입니다. 가능한 최고의 사용자 경험을 제공하기 위해 앱에 포함할 내용을 신중하게 선택해야 합니다.

## 암묵적 합의

이슈를 승인하면 다음을 약속합니다:

- **작업이 시기적절하게 주목받을 것이며**, 차단되지 않도록 다른 우선순위 항목을 미룰 것입니다.
- 기능이 잘 완성될 수 있도록 **엔지니어링 품질, 시스템 설계 등에 대한 코칭**을 제공할 것입니다.
- Onyx 팀이 기능을 병합에 필요한 품질로 구축하는 데 필요한 **모든 리소스를 갖출 수 있도록** 디자인, PM, 엔지니어링에서 리소스와 여력을 확보할 것입니다.

이것은 팀의 큰 투자이므로, 다음을 요청합니다:

- 디자인 문서, 엔지니어링 Best Practice의 모든 요구 사항을 철저히 읽고, Onyx 팀의 오버헤드를 최소화하려고 노력하세요.
- 컨텍스트 전환과 Onyx 팀의 지속적인 리소스 소비를 줄이기 위해 **기능을 시기적절하게 완성**하세요.
